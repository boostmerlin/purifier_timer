# 云端同步功能说明

## 功能概述

使用 TextDB.online 免费云端存储服务,实现跨设备数据同步。

## 主要功能

### 1. 启用云端同步

**方式一: 生成新密钥**
- 点击"云端同步"按钮
- 选择"生成新密钥"
- 系统自动生成20位随机密钥
- 妥善保管密钥,可在其他设备使用

**方式二: 使用已有密钥**
- 点击"云端同步"按钮
- 选择"使用已有密钥"
- 输入在其他设备上生成的密钥
- 连接后即可访问相同数据

### 2. 自动同步

- 数据修改后自动同步到云端
- 本地缓存确保离线可用
- 启动时优先从云端加载最新数据

### 3. 手动拉取

- 点击"拉取云端数据"
- 强制从云端获取最新数据
- 覆盖本地数据

### 4. 禁用同步

- 禁用后数据仅保存本地
- 云端数据不会被删除
- 可随时重新启用

## 技术细节

### 数据流程

1. **初始化**
   - 检查本地是否保存云端 key
   - 有 key: 优先从云端加载 → 失败则从本地加载
   - 无 key: 从本地加载 → 无数据则使用默认数据

2. **数据保存**
   - 每次修改自动保存到本地 (AsyncStorage)
   - 如果启用云端同步,同时上传到云端
   - 防抖处理避免频繁请求

3. **多设备同步**
   - 设备 A 生成密钥并上传数据
   - 设备 B 使用相同密钥
   - 两设备共享同一云端数据

### API 限制

- 读取次数不限
- 每个 IP 每日更新限制 500 次
- 数据 1 年无更新自动删除
- 单条数据最大 20 万字符

### 安全性

- 密钥长度 29 位 (purifier_ + 20位随机)
- 碰撞概率极低
- 无密码保护,请勿存储敏感信息
- 建议使用完毕后手动删除云端数据

## 使用建议

1. **首次使用**: 生成新密钥
2. **其他设备**: 使用已有密钥连接
3. **定期备份**: 复制密钥保存
4. **数据迁移**: 使用密钥在新设备恢复
5. **停用后**: 可选择删除云端数据

## 故障排除

### 同步失败
- 检查网络连接
- 验证密钥格式正确
- 查看控制台错误日志

### 数据不一致
- 点击"拉取云端数据"强制同步
- 确保所有设备使用相同密钥

### 无法连接
- 检查 TextDB.online 服务状态
- 验证 API 请求限制未超额
- 尝试禁用后重新启用
